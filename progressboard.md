- **Fixed window-raising for closed and full-screen/cross-Space windows** (`daemon/actions.py`, [#1](https://github.com/ictashik/AgentDeck/issues/1)). `_raise_vscode_window()`'s only mechanism was System Events (exact-title match + `AXRaise`), which can't enumerate windows outside the current macOS Space at all — so a bound repo whose window had been closed, or whose window was full-screened into its own Space, could never be raised; the blanket-`activate` fallback only helped by coincidence (it brings forward whichever window is the app's own last-active one, not necessarily the target repo's). Added a fallback, tried only when AXRaise can't find/raise a title match: `open -a "Visual Studio Code" <repo>`. This is not `-r` — it hands the path to VS Code's own running instance over LaunchServices/IPC and lets VS Code itself decide (open a fresh window if the repo isn't open anywhere; request focus on its own existing window if it is), so macOS permits the Space switch since the app is asking for its own window, unlike a background process forcing it via System Events. Live-verified both cases on real windows: reopened a closed repo's window, and raised a specific full-screen repo's window from a different Space while another window was the app's currently-active one (confirming it targets the right window, not just whatever's already frontmost).
